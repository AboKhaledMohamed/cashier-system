## ⚠️ نقاط الضعف المحتملة

### 1. الاستعلامات الثقيلة (Heavy Queries)

```sql
-- ⚠️ تقرير المنتجات المباعة - يمكن أن يكون بطيء مع بيانات كثيرة
SELECT ii.product_name, SUM(ii.qty) as total_qty, SUM(ii.total) as total_revenue
FROM invoice_items ii
JOIN invoices i ON ii.invoice_id = i.id
WHERE i.date = ? AND i.invoice_type = 'بيع' AND i.status = 'مكتمل'
GROUP BY ii.product_name ORDER BY total_revenue DESC LIMIT 10
```
**التحسين المقترح:** Composite index على `(invoices.date, invoices.invoice_type, invoices.status)`

---

### 2. جلب كل البيانات (Full Table Scans)

```javascript
// returns.js - يجلب 500 سجل مع كل الـ items
const returns = db.prepare('SELECT * FROM returns ORDER BY created_at DESC LIMIT 500').all();
for (const ret of returns) {
  ret.items = db.prepare('SELECT * FROM return_items WHERE return_id = ?').all(ret.id);
}
```
**المشكلة:** N+1 queries - استعلام منفصل لكل مرتجع

**التحسين:**
```sql
-- استعلام واحد مع JOIN
SELECT r.*, ri.* 
FROM returns r 
LEFT JOIN return_items ri ON r.id = ri.return_id 
ORDER BY r.created_at DESC 
LIMIT 500
```

---

### 3. التقارير الشهرية

```sql
-- تقرير المبيعات الشهرية
SELECT * FROM invoices 
WHERE date BETWEEN ? AND ? AND invoice_type = 'بيع' AND status = 'مكتمل'
ORDER BY date, time
```
**التحسين:** Index مركب `(date, invoice_type, status)`

---

### 4. حركات المخزون

```sql
-- جلب كل حركات منتج معين
SELECT * FROM stock_movements WHERE product_id = ? ORDER BY created_at DESC
```
**التحسين:** Composite index `(product_id, created_at DESC)`

## 🔧 التحسينات المقترحة

### 1. إضافة Indexes مركبة (Composite Indexes)

```sql
-- للتقارير اليومية
CREATE INDEX IF NOT EXISTS idx_invoices_report 
ON invoices(date, invoice_type, status);

-- لسجل العمليات
CREATE INDEX IF NOT EXISTS idx_audit_date_action 
ON audit_log(created_at, action);

-- للمدفوعات
CREATE INDEX IF NOT EXISTS idx_payments_date_party 
ON payments(date, party_type, payment_direction);

-- لحركات المخزون
CREATE INDEX IF NOT EXISTS idx_stock_mov_product_date 
ON stock_movements(product_id, created_at DESC);
```

### 2. تحسين LIMIT مع ORDER BY

```sql
-- بدلاً من: SELECT * FROM table ORDER BY date LIMIT 500
-- استخدم: SELECT * FROM table ORDER BY date DESC LIMIT 500
-- مع index على (date DESC)
```

### 3. تجنب SELECT *

```javascript
// ❌ بطيء
const data = db.prepare('SELECT * FROM large_table').all();

// ✅ أسرع - اختر الأعمدة المطلوبة فقط
const data = db.prepare('SELECT id, name, price FROM large_table').all();
```

## 📈 مراقبة الأداء

### 1. تفعيل Query Logging (للتطوير فقط)

```javascript
// في database.js
const db = new Database(dbPath, {
  verbose: process.env.NODE_ENV === 'development' ? console.log : null
});
```

### 2. وقت تنفيذ الاستعلامات

```javascript
// Wrapper لقياس الوقت
const timedQuery = (db, sql, params) => {
  const start = performance.now();
  const result = db.prepare(sql).all(...params);
  const duration = performance.now() - start;
  if (duration > 100) { // أكثر من 100ms
    console.warn(`Slow query (${duration.toFixed(2)}ms): ${sql}`);
  }
  return result;
};
```
