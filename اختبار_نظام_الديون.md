# خطة اختبار نظام الديون - Debt System Test Plan

## ✅ سيناريوهات الاختبار

### السيناريو 1: بيع آجل جديد
```
الخطوات:
1. اختر عميل له دين قديم = 500 جنيه
2. أنشئ فاتورة آجلة بقيمة 300 جنيه (دفع جزئي 100، دين 200)
3. تحقق من:
   ✓ الفاتورة المطبوعة: إجمالي الدين = 700 (500+200)
   ✓ صفحة العملاء: الدين الحالي = 700
   ✓ database: current_balance = 700
   ✓ Dashboard: total_customer_debt متحدث
```

**النتيجة المتوقعة:**
- `current_balance` = 700
- `total_debts` = 700 (أو تراكمي)
- `credit_used` = 700

---

### السيناريو 2: تحصيل دين من العميل
```
الخطوات:
1. اختر عميل له دين = 700 جنيه
2. سجل دفعة "تحصيل" بقيمة 300 جنيه
3. تحقق من:
   ✓ صفحة العملاء: الدين الحالي = 400
   ✓ database: current_balance = 400
   ✓ سجل الدفعات موجود
```

**النتيجة المتوقعة:**
- `current_balance` = 400 (700 - 300)
- `credit_used` = 400
- `total_paid` += 300
- `last_payment_date` = today

---

### السيناريو 3: إلغاء فاتورة آجلة
```
الخطوات:
1. أنشئ فاتورة آجلة بقيمة 200 جنيه
2. تحقق من زيادة الدين
3. ألغِ الفاتورة
4. تحقق من:
   ✓ عكس الدين (خصم 200 من الدين)
   ✓ إرجاع المخزون
   ✓ تحديث إحصائيات الـ Dashboard
```

**النتيجة المتوقعة:**
- `current_balance` يقل بـ 200
- `MAX(0, ...)` يضمن عدم السالب

---

### السيناريو 4: دفع نقدي (لا يؤثر على الدين)
```
الخطوات:
1. اختر عميل له دين قديم = 500
2. أنشئ فاتورة نقدية بقيمة 300 (دفع كامل)
3. تحقق:
   ✓ لا يزيد الدين
   ✓ يزيد total_purchases فقط
```

**النتيجة المتوقعة:**
- `current_balance` = 500 (ثابت)
- `total_purchases` += 300
- `last_purchase_date` = today

---

### السيناريو 5: دفع جزئي نقدي + آجل
```
الخطوات:
1. فاتورة بقيمة 500 جنيه
2. دفع نقدي: 200، دين: 300
3. تحقق:
   ✓ الـ cash_session: total_sales_cash += 200
   ✓ الـ cash_session: total_sales_deferred += 300
   ✓ الدين يزيد بـ 300 فقط
```

---

### السيناريو 6: ديون الموردين (Suppliers)
```
الخطوات:
1. إضافة مورد بدين افتتاحي
2. تسجيل مشتريات آجلة
3. تسجيل دفع للمورد
4. تحقق:
   ✓ current_balance يقل
   ✓ تحديث حالة المشتريات (مدفوع/جزئي)
```

## 🔍 نقاط التحقق في كل صفحة

### صفحة البيع (POSPage.tsx)
| العنصر | التحقق |
|--------|--------|
| `customer_total_debt` | `(current_balance || 0) + credit_amount` |
| إظهار الدين | في الإيصال المطبوع فقط إذا كان credit > 0 |

### صفحة العملاء (CustomersPage.tsx)
| العنصر | التحقق |
|--------|--------|
| `totalDebt` | `customers.reduce((sum, c) => sum + (c.current_balance || 0), 0)` |
| عرض العميل | `formatCurrency(customer.current_balance \|\| 0)` |

### صفحة الموردين (SuppliersPage.tsx)
| العنصر | التحقق |
|--------|--------|
| `totalDebt` | `suppliers.reduce((sum, s) => sum + ((s as any).current_balance \|\| s.debt_used) \|\| 0), 0)` |
| دفع للمورد | `api.payments.create({ payment_direction: 'سداد', party_type: 'supplier' })` |

### لوحة التحكم (Dashboard)
| العنصر | المصدر |
|--------|--------|
| `total_customer_debt` | `SELECT COALESCE(SUM(current_balance), 0) FROM customers WHERE is_active = 1` |
| `total_supplier_debt` | `SELECT COALESCE(SUM(current_balance), 0) FROM suppliers WHERE is_active = 1` |
| `today_collections` | `SELECT COALESCE(SUM(amount), 0) FROM payments WHERE date = today AND party_type = 'customer' AND payment_direction = 'تحصيل'` |

## ⚠️ الحالات الحدية (Edge Cases)

| الحالة | السلوك المتوقع |
|--------|----------------|
| إلغاء فاتورة دين أكبر من الدين الحالي | `MAX(0, current_balance - amount)` |
| دفع أكبر من الدين | `MAX(0, current_balance - amount)` |
| عميل محذوف (is_active=0) | لا يظهر في الإحصائيات |
| فاتورة بقيمة صفر | لا تؤثر على الدين |

## 🔧 SQL للتحقق يدوياً

```sql
-- ديون كل العملاء النشطين
SELECT id, name, current_balance, credit_used, total_debts, total_paid 
FROM customers WHERE is_active = 1 AND current_balance > 0;

-- إجمالي ديون العملاء
SELECT COALESCE(SUM(current_balance), 0) as total_customer_debt 
FROM customers WHERE is_active = 1;

-- فواتير آجلة اليوم
SELECT invoice_number, customer_name, credit_amount, date 
FROM invoices 
WHERE payment_method = 'آجل' AND date = date('now');

-- تحصيلات اليوم
SELECT party_name, amount, date 
FROM payments 
WHERE party_type = 'customer' AND payment_direction = 'تحصيل' AND date = date('now');

-- ديون الموردين
SELECT id, name, current_balance FROM suppliers WHERE is_active = 1 AND current_balance > 0;
```
