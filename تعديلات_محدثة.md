## اختبار نظام الديون

### 🎯 الهدف
اختبار نظام الديون في جميع الصفحات والتأكد من:
- ✅ حساب الديون بشكل صحيح
- ✅ تحديث الديون تلقائياً
- ✅ عرض الديون في كل الصفحات المتعلقة
- ✅ عكس الديون عند الإلغاء

### 📋 قائمة الصفحات المُختبرة

#### 1. صفحة البيع (POSPage.tsx)
**العناصر المختبرة:**
- ✅ عرض العميل المختار مع دينه الحالي
- ✅ إضافة دين جديد عند البيع الآجل
- ✅ حساب إجمالي الدين في الفاتورة المطبوعة
- ✅ تحذير تجاوز الحد الائتماني (إن وجد)

**سيناريوهات الاختبار:**
```
✅ 1.1 بيع آجل كامل (لا يوجد دفع نقدي)
    - قبل: current_balance = 500
    - فاتورة: 300 (credit_amount = 300)
    - بعد: current_balance = 800
    - الفاتورة المطبوعة: إجمالي الدين = 800 ✓

✅ 1.2 بيع آجل جزئي (دفع نقدي + دين)
    - قبل: current_balance = 500
    - فاتورة: 500 (paid = 200, credit_amount = 300)
    - بعد: current_balance = 800
    - الفاتورة: مبلغ الدين = 300، إجمالي الدين = 800 ✓

✅ 1.3 بيع نقدي (لا يزيد الدين)
    - قبل: current_balance = 500
    - فاتورة: 400 (نقدي كامل)
    - بعد: current_balance = 500 (ثابت)
    - الفاتورة: لا يظهر إجمالي الدين ✓
```

#### 2. صفحة العملاء (CustomersPage.tsx)
**العناصر المختبرة:**
- ✅ عرض الدين الحالي لكل عميل
- ✅ إجمالي الديون في الإحصائيات
- ✅ حذف العميل وديونه

**سيناريوهات الاختبار:**
```
✅ 2.1 عرض الدين في الجدول
    - كل عميل يظهر current_balance ✓
    - التحديث فوري بعد البيع الآجل ✓

✅ 2.2 إجمالي الديون في الإحصائيات
    - sum(current_balance) لكل العملاء ✓
    - تحديث فوري ✓

✅ 2.3 حذف عميل له ديون
    - soft delete مع الاحتفاظ بالديون للتقارير
```

#### 3. صفحة الموردين (SuppliersPage.tsx)
**العناصر المختبرة:**
- ✅ عرض ديون الموردين
- ✅ سداد دين المورد
- ✅ إضافة دين عند شراء آجل

**سيناريوهات الاختبار:**
```
✅ 3.1 شراء آجل من مورد
    - قبل: current_balance = 1000
    - فاتورة شراء: 500
    - بعد: current_balance = 1500 ✓

✅ 3.2 سداد دين للمورد
    - قبل: current_balance = 1500
    - دفعة: 500
    - بعد: current_balance = 1000 ✓

✅ 3.3 عرض في الجدول
    - debt_used أو current_balance يظهر بشكل صحيح ✓
```

#### 4. لوحة التحكم (DashboardPage.tsx)
**العناصر المختبرة:**
- ✅ إجمالي ديون العملاء (total_customer_debt)
- ✅ إجمالي ديون الموردين (total_supplier_debt)
- ✅ تحصيلات اليوم (today_collections)

#### 5. صفحة المرتجعات (ReturnsPage.tsx)
**العناصر المختبرة:**
- ✅ إرجاع فاتورة آجلة وعكس الدين
- ✅ إرجاع جزئي

**سيناريوهات الاختبار:**
```
✅ 6.1 مرتجع كامل لفاتورة آجلة
    - قبل: current_balance = 800
    - فاتورة آجلة: 300
    - مرتجع كامل: -300
    - بعد: current_balance = 500 ✓

✅ 6.2 مرتجع جزئي
    - فاتورة آجلة: 500
    - مرتجع: 200
    - الدين يقل بـ 200 فقط ✓
```

#### 6. صفحة المدفوعات (PaymentsPage)
**العناصر المختبرة:**
- ✅ تسجيل تحصيل من عميل
- ✅ تسجيل سداد لمورد
- ✅ تأثيره على الأرصدة

**سيناريوهات الاختبار:**
```
✅ 7.1 تحصيل من عميل
    - قبل: current_balance = 800
    - تحصيل: 300
    - بعد: current_balance = 500 ✓
    - UPDATE: credit_used -= 300 ✓

✅ 7.2 سداد لمورد
    - قبل: current_balance = 1500
    - سداد: 500
    - بعد: current_balance = 1000 ✓
```

### 🔗 علاقات الديون بين الصفحات

```
┌─────────────────────────────────────────────────────────────┐
│                      صفحة البيع (POS)                       │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  • اختيار العميل → يظهر current_balance             │   │
│  │  • إتمام بيع آجل → يزيد current_balance            │   │
│  │  • الفاتورة المطبوعة → تعرض إجمالي الدين           │   │
│  └────────────────────┬─────────────────────────────────┘   │
└───────────────────────┼─────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────────┐
│                   صفحة العملاء (Customers)                   │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  • الجدول يعرض current_balance لكل عميل              │   │
│  │  • الإحصائيات: إجمالي الديون                        │   │
│  │  • حذف العميل → soft delete (يحتفظ بالديون)        │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────────┐
│                   لوحة التحكم (Dashboard)                    │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  • total_customer_debt: SUM(current_balance)         │   │
│  │  • today_collections: SUM(payments اليوم)          │   │
│  │  • إحصائيات المبيعات الآجلة                          │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────────┐
│                   صفحة المدفوعات (Payments)                  │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  • تحصيل من عميل → ينقص current_balance              │   │
│  │  • سداد لمورد → ينقص current_balance للمورد        │   │
│  │  • سجل كل العمليات                                  │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### ✅ SQL للتحقق من الاختبار

```sql
-- التحقق من ديون عميل معين قبل وبعد
SELECT id, name, current_balance, credit_used, total_debts, total_paid 
FROM customers WHERE id = 'customer-id';

-- التحقق من فواتير العميل الآجلة
SELECT invoice_number, date, total, credit_amount, status 
FROM invoices 
WHERE customer_id = 'customer-id' AND payment_method = 'آجل';

-- التحقق من تحصيلات العميل
SELECT amount, date, notes 
FROM payments 
WHERE party_id = 'customer-id' AND party_type = 'customer' AND payment_direction = 'تحصيل';

-- التحقق من إجمالي ديون الموردين
SELECT SUM(current_balance) as total_supplier_debt 
FROM suppliers WHERE is_active = 1;
```
